## 1. プロジェクト概要

### 何をしたか

富士山を背景にゴルファーがスイングしている写真（5000×3333px、300DPI、Adobe RGB）において、逆光で暗くなった**顔の部分だけ**を自然に明るく補正しました。

### 最も重要な成果

- **背景の99.13%のピクセルが元画像と完全に同一**（1ビットも変わっていない）
- 顔の暗い領域だけをピンポイントで補正（画像全体の**わずか0.3%**の領域）
- ICCカラープロファイル（Adobe RGB）、EXIF情報、DPI（300）をすべて完全保持
- 処理はすべて自動 ― 人間の操作は「スクリプトを実行する」だけ

---

## 2. 課題と解決アプローチ

### 直面した課題

| 課題 | 内容 | 解決策 |
|------|------|--------|
| 顔の検出精度 | 横顔＋帽子＋逆光で従来の顔検出（OpenCV YuNet、Haar）が失敗 | AIセマンティックセグメンテーション（SegFormer）に切替 |
| 不自然な明るさ | 単純な輝度アップは「顔が白く光る」不自然な結果に | DaVinci Resolve方式のLift/Gamma/Offset微調整 |
| 背景の色味変化 | LAB色空間の丸め誤差で背景ピクセルが微妙に変わる | RGB空間での元画像直接ブレンド |
| 背景の鮮やかさ低下 | ICCカラープロファイル（Adobe RGB）が出力時に欠落 | ICC/EXIF/DPIの完全保持 |
| マスクの漏れ | ガウシアンブラーが遠方まで影響を広げる | パワーカーブ＋二段階閾値カット |

### アプローチの変遷

```
試行1: OpenCV YuNet顔検出 → 横顔で検出失敗
試行2: DaVinci Resolve手動操作 → 解像度制限・DPI低下・背景色変化
試行3: SegFormer AIセグメンテーション + DaVinci Resolve式グレーディング → 成功
```

## 3. 処理パイプライン（非エンジニア向け）

### Step 1: AIが顔のパーツを認識する

人間の目が「ここが肌、ここが眉、ここが唇」と認識するのと同じように、AIモデル（SegFormer）が写真の中の顔のパーツをピクセル単位で識別します。

このAIは**19種類の顔パーツ**（肌、左眉、右眉、左目、右目、鼻、口、上唇、下唇、左耳、右耳、眼鏡、帽子、首、ネックレス、服、髪など）を区別できます。横顔であっても、帽子をかぶっていても、逆光で暗くても正確に認識します。

今回の写真では以下が検出されました：
- 肌: 0.58%
- 帽子: 1.92%
- 右眉: 0.64%
- 下唇: 0.76%

### Step 2: 「暗い顔の肌」だけを選ぶマスクを作る

AIが認識した「肌」の領域に対して、さらに**暗さのフィルター**をかけます。明るい肌（例えば帽子のツバで影になっていない部分）はそのまま、暗い肌だけが補正対象になります。

これは、DaVinci Resolveで「クオリファイアー（Qualifier）」を使って肌色を選択し、さらにルミナンス（明るさ）で暗い部分だけに絞り込む操作と同じ考え方です。

最終的に、画像全体のうち**たった0.3%の領域**だけが補正対象として選ばれました。

### Step 3: プロのカラリストと同じ方法で色を調整する

映画やCMの色調整で使われるDaVinci Resolveの**カラーホイール**と同じ3つの調整を行います：

| 調整 | DaVinci Resolveの対応機能 | 何をしているか | 設定値 |
|------|---------------------------|----------------|--------|
| **Lift** | シャドウホイール | 暗い部分を少し持ち上げる | 自動算出（今回: +0.0598） |
| **Gamma** | ミッドトーンホイール | 中間の明るさを微調整 | 自動算出（今回: +0.0452） |
| **Offset** | オフセット | 全体をほんの少し持ち上げ | 自動算出（今回: +0.0116） |
| **彩度** | サチュレーション | 色の鮮やかさを微補正 | 自動算出（今回: +5.8%） |

これらの値は**顔領域の輝度ヒストグラムを分析して自動で算出**されます。画像が暗いほどLiftが強く、明るい画像では控えめになります。DaVinci Resolveのカラリストが写真を見て「このくらい」と判断するのと同じ判断をコンピュータが自動で行います。

### Step 4: 肌色の方向を守る

カラリストがベクトルスコープを見ながら「スキントーンライン（I-Line）」から外れないように調整するのと同じ処理をコンピュータが自動で行います。肌の色が赤すぎたり黄色すぎたりしないよう、元の肌色の方向を維持します。

### Step 4.5: 肌を自然に滑らかにする

SegFormerが識別した「肌」の領域だけに対して、**バイラテラルフィルタ**（エッジを保持しつつ滑らかにするフィルタ）を適用します。目・眉・唇はマスクから除外されているため、肌以外のパーツはまったく影響を受けません。

これは美肌アプリのような「塗りつぶし」ではなく、**肌のキメを少しだけ整える程度の自然な処理**です。強度は0.4（最大1.0）に設定しており、元の肌のテクスチャを残しつつ、逆光で暗かった部分を明るくしたときに目立ちやすいノイズや肌荒れを目立たなくします。

### Step 5: 元画像を壊さずに保存する

- **マスク外（背景）のピクセルは元画像のデータをそのままコピー** ― LAB変換の丸め誤差すらゼロ
- Adobe RGBカラープロファイルを完全保持
- EXIF（カメラ情報、撮影日時、レンズ情報など）を完全保持
- 300 DPIを完全保持
- TIFF（完全無劣化）とJPEG（品質98）の両方で出力

## 4. 技術詳細（エンジニア向け）

### 4.1 使用技術スタック

| 技術 | バージョン/詳細 | 役割 |
|------|-----------------|------|
| Python | 3.x | スクリプト言語 |

| Pillow (PIL) | JPEG/TIFF I/O、EXIF/ICC保持 | 画像入出力 |

### 4.2 AIモデル: SegFormer (jonathandinu/face-parsing)

- **アーキテクチャ**: SegFormer (MIT-B0〜B5ベース)
- **学習データ**: CelebAMask-HQ（30,000枚の顔画像に19クラスのピクセルレベルアノテーション）
- **出力**: 19クラスのセマンティックセグメンテーションマップ + クラス確率マップ
- **推論**: GPU (CUDA) またはCPU対応。5000×3333の画像で数秒（GPU）

```python
# SegFormerの出力はロジット → ソフトマックスでクラス確率を取得
probs = torch.softmax(upsampled, dim=1)[0]  # shape: (19, H, W)
labels = upsampled.argmax(dim=1)[0]          # shape: (H, W)
```

確率マップを使うことで、ハードなバイナリマスクではなく**ソフトなグラデーションマスク**を生成できます。これが自然な境界遷移の鍵です。

### 4.3 マスク生成パイプライン

```
SegFormer確率マップ
    ↓ クラス重み付け合成（肌:1.0, 顔パーツ:0.9, 首:0.7）
raw_mask (0〜1の連続値)
    ↓ × darkness_weight = (1 - L/255)^1.2
    ↓   → 暗いピクセルほど重みが大きい
final_mask（暗い肌のみ）
    ↓ 閾値カット < 0.10 → 0（ブラー前のテール除去）
    ↓ GaussianBlur(ksize=11)（アンチエイリアス）
    ↓ 正規化 → max=1.0
    ↓ パワーカーブ x^2.0（低値の圧縮、テール除去）
    ↓ 閾値カット < 0.03 → 0（ブラー後の残留除去）
最終マスク（0.30%の精密領域）
```

**パワーカーブ `x^2.0` の効果**:
- mask=0.1 → 0.01（実質ゼロに）
- mask=0.3 → 0.09（ほぼゼロに）
- mask=0.5 → 0.25（緩やかに効く）
- mask=0.9 → 0.81（ほぼそのまま）

これにより、ガウシアンブラーで広がった低い値のテールが効果的に除去され、背景への漏れを防ぎます。

### 4.4 DaVinci Resolve方式のカラーグレーディング

LAB色空間のLチャンネル（輝度）に対して、3つの独立したトーンカーブを順次適用：

```python
# Lift: シャドウ領域に選択的に作用
influence = (1.0 - l_norm) ** 0.8  # 暗いほど影響大
l_new = l_norm + amount * influence

# Gamma: ミッドトーンに選択的に作用
influence = 4.0 * l_norm * (1.0 - l_norm)  # ベルカーブ（中央がピーク）
l_new = l_norm + amount * influence

# Offset: 全トーンに均一に作用
l_new = l_norm + amount
```

処理順序は `Lift → Gamma → Offset`（DaVinci Resolveの内部処理と同一）。

### 4.5 Lift/Gamma/Offsetの自動算出（実装済み）

画像ごとに最適なグレーディングパラメータを自動算出する3段階のアルゴリズム：

**Step A: 顔領域の輝度分析**
```python
stats = analyze_face_luminance(l_channel, mask)
# → mean, median, p10, p90, std を計算
```

**Step B: 目標輝度の決定**
```python
# 暗い画像ほど大きく補正、明るい画像ほど控えめに
if current < 40:    target = min(current * 1.7, current + 40)   # 非常に暗い
elif current < 70:  target = min(current * 1.45, current + 30)  # 暗い（逆光）
elif current < 100: target = min(current * 1.2, current + 18)   # やや暗い
else:               target = current                             # 補正不要
```

**Step C: パラメータの逆算**
```python
intensity = min(gap / 30.0, 1.0)  # gap = target - current
shadow_ratio = max(0, min(1, (70 - mean) / 70))

lift   = intensity * (0.055 + 0.095 * shadow_ratio)  # シャドウ重視
gamma  = intensity * (0.028 + 0.050 * mid_ratio)      # ミッドトーン重視
offset = intensity * 0.020                              # 常に控えめ
sat    = intensity * 0.10 * (1 - min(std/30, 0.5))     # 彩度低いほど強く
```

### 4.6 肌スムージング（バイラテラルフィルタ）

SegFormerの「肌」クラス（クラス1）のみを使用し、目・眉・唇を除外したマスクを生成。このマスク内にのみバイラテラルフィルタを適用：

```python
# バイラテラルフィルタ: エッジ（輪郭）を保持しつつ平面を滑らかに
d = max(5, min(15, max(h, w) // 500))       # フィルタ径（画像サイズに応じて自動調整）
sigma_color = 40 + 30 * strength             # 色の類似度（strength=0.4で52）
sigma_space = 40 + 30 * strength             # 空間的な広がり
smoothed = cv2.bilateralFilter(img, d, sigma_color, sigma_space)

# マスクブレンド（肌領域のみ適用、微小値カット0.05で背景保護）
result = img * (1 - mask * strength) + smoothed * (mask * strength)
```

バイラテラルフィルタはガウシアンブラーと異なり、**色の変化が大きいエッジ（目元、鼻筋、唇のライン）を保持**しつつ、肌の平面部分のみを滑らかにします。これにより「塗りつぶし感」のない自然な仕上がりになります。

### 4.7 スキントーンベクトル保持（I-Line維持）

LAB空間のa（赤-緑軸）とb（黄-青軸）チャンネルにおいて、彩度変更のデルタベクトルを**元の肌色方向に射影**します：

```python
# 元の肌色の平均方向（ベクトルスコープ上の角度）
skin_angle = arctan2(mean_b, mean_a)  # 今回: 68.3°

# 色変更のデルタを肌色方向に射影
projection = a_delta * cos(skin_angle) + b_delta * sin(skin_angle)

# 肌色方向に沿ったデルタのみを適用
a_corrected = a_orig + projection * cos(skin_angle) * mask
b_corrected = b_orig + projection * sin(skin_angle) * mask
```

これにより、彩度を変更しても色相がスキントーンライン（I-Line）から逸脱しません。

### 4.8 背景保護メカニズム

```python
# マスク外のピクセルは元画像のRGB値をそのまま使用
# LAB変換の丸め誤差は一切発生しない
orig_rgb = np.array(pil_image)           # 元のRGB（無変換）
graded_rgb = cv2.cvtColor(...)            # グレーディング後のRGB
final_rgb = orig_rgb * (1 - mask) + graded_rgb * mask
```

マスク値が0.0の領域は `orig_rgb * 1.0 + graded_rgb * 0.0 = orig_rgb` となり、**数学的に元画像と同一**です。

## 5. 工夫したポイント

### 5.1 なぜSegFormerか

| 方式 | 精度 | 横顔対応 | 逆光対応 | 帽子対応 |
|------|------|----------|----------|----------|
| OpenCV Haar | 低 | 一部 | 弱い | 弱い |
| OpenCV YuNet DNN | 中 | 対応 | やや弱い | やや弱い |
| **SegFormer** | **高** | **完全対応** | **強い** | **完全対応** |

SegFormerはピクセル単位のセマンティックセグメンテーションであるため、「顔の**どの部分**が肌か」をピクセルレベルで判定できます。バウンディングボックス（四角形）ではなく、実際の肌の形状に沿ったマスクが得られます。

### 5.2 暗さの重み付け

単にSegFormerの肌マスクを使うだけでは、帽子のツバで影になっていない明るい肌まで補正対象になってしまいます。`darkness_weight = (1 - L/255)^1.2` を掛けることで、**暗い肌だけ**に絞り込みます。

### 5.3 パワーカーブによるテール除去

ガウシアンブラーは数学的に無限遠まで値を広げます。閾値カットだけでは不十分で、ブラー後に残る微小な値（0.01〜0.05程度）が大量のピクセルに1〜2段階の色変化を引き起こします。`x^2.0` のパワーカーブで低い値を非線形に圧縮することで、エッジの自然さを保ちつつテールを効果的に除去しました。

### 5.4 ICCプロファイルの保持

これが最も見落としやすいポイントでした。元画像がAdobe RGB（sRGBより広い色域）で保存されている場合、出力にICCプロファイルを含めないと、ビューアがsRGBとして解釈します。Adobe RGBの色をsRGBとして表示すると、彩度が落ちてのっぺりした印象になります。

## 7. SaaS展開の可能性

### 7.1 直接的なSaaSプロダクト案

#### A. ウェディング・イベント写真の一括補正サービス

**概要**: ウェディングフォトグラファーやイベントカメラマンが、逆光や影のある大量の写真を一括で自動補正するクラウドサービス。

**市場**: ウェディングフォト業界では1回の撮影で500〜2000枚の写真を扱います。そのうち20〜30%は逆光や影の問題を含んでいます。手動で1枚ずつ補正するのは膨大な作業です。

**価値**:
- 手動作業: 1枚あたり3〜10分 × 数百枚 = 数十時間
- 本ツール: 1枚あたり数秒（GPU使用時）× 数百枚 = 数十分
- **作業時間を1/100以下に短縮**

#### B. ECサイト向け商品写真の人物補正

**概要**: アパレルECのモデル写真やポートレートの肌補正を自動化。

**価値**: 大量のSKUを持つアパレルブランドは、毎シーズン数千枚のモデル写真を撮影します。均一な肌の明るさ・色味の統一が求められますが、手動では品質にバラつきが出ます。

#### C. SNS向けポートレート自動補正API

**概要**: スマートフォンアプリやSNSプラットフォームに組み込むAPI。ユーザーがアップロードした写真の顔を自動で最適化。

**差別化**: 既存の「美肌フィルター」は肌を塗りつぶして不自然にしますが、本技術はプロのカラリストと同じ手法で**自然な補正**を行います。

#### D. 不動産・建築写真の室内補正

**概要**: SegFormerのモデルを室内オブジェクト向けに変更すれば、不動産写真の窓辺の逆光問題（室内が暗い）を自動補正できます。

#### 9.2 ワークフロー面

| 項目 | 手動 | Python自動 |
|------|------|-----------|
| 処理時間/枚 | 8〜12分 | **3〜10秒** |
| 解像度制限 | DaVinci Resolve無料版: 3840×2160まで | **制限なし** |
| DPI保持 | DaVinci Resolve: 96DPIに低下 | **完全保持** |
| ICCプロファイル | 設定を間違えると色が変わる | **自動で完全保持** |

#### 9.3 今回実際に発生した問題

DaVinci Resolve（無料版）で手動補正を試みた際に発生した問題：

1. **解像度制限**: 5000×3333の画像を読み込むと、3840×2160に自動リサイズされた
2. **DPI低下**: 300DPIが96DPIに変更された（印刷品質に影響）
3. **背景の色変化**: パワーウィンドウで顔だけにグレーディングを適用しても、DaVinci Resolveの内部カラーマネジメント（ACESやDaVinci WGなど）が書き出し時に全体に影響した
4. **ICCプロファイル**: Adobe RGBの画像がsRGBとして扱われ、色域が変わった

本Python処理ではこれらの問題はすべて発生しません。

## 10. 自動適応パラメータ設計（実装済み）

### 10.1 適応的パラメータの全体像

本システムでは、**すべての主要パラメータが画像ごとに自動で適応**します。固定値に依存する要素はほとんどありません。

| 要素 | 状態 | 仕組み |
|------|------|--------|
| スキントーン角度 | 適応的 | `arctan2(mean_b, mean_a)` で実際の肌色から自動算出 |
| マスク形状 | 適応的 | SegFormer推論で写真ごとにピクセル単位で認識 |
| 暗さの重み | 適応的 | `(1 - L/255)^1.2` で輝度分布に基づき自動調整 |
| ベクトル射影方向 | 適応的 | スキントーン角度に連動 |
| **Lift/Gamma/Offset** | **適応的（実装済み）** | 顔領域の輝度ヒストグラムから自動算出 |
| **彩度補正** | **適応的（実装済み）** | 輝度・標準偏差から自動算出 |
| **肌スムージング** | **適応的（実装済み）** | SegFormerの肌マスクでピクセル単位で適用 |
| マスク閾値 | 固定（汎用的） | 0.10 / 0.03（ほとんどの写真で有効に機能） |
| darkness指数 | 固定（汎用的） | 1.2（汎用的なパラメータ） |

### 10.2 自動グレーディングアルゴリズム（実装済み）

#### 3段階の自動算出パイプライン

**Step A: 顔領域の輝度分析** (`analyze_face_luminance()`)

```python
face_region = mask > 0.3
stats = {
    'mean':   float(np.mean(face_l)),           # 平均輝度
    'median': float(np.median(face_l)),          # 中央値
    'p10':    float(np.percentile(face_l, 10)),  # 暗い側10%
    'p90':    float(np.percentile(face_l, 90)),  # 明るい側90%
    'std':    float(np.std(face_l)),             # 標準偏差
}
```

**Step B: 目標輝度の決定** (`calculate_target_luminance()`)

暗い画像ほど大きく補正し、明るい画像ほど控えめにする段階的な目標設定：

| 現在の平均輝度 | 目標計算 | 例 |
|---------------|---------|-----|
| < 40（非常に暗い） | `min(current * 1.7, current + 40)` | 30 → 51 |
| 40〜70（暗い/逆光） | `min(current * 1.45, current + 30)` | 44 → 63.8 |
| 70〜100（やや暗い） | `min(current * 1.2, current + 18)` | 80 → 96 |
| > 100（十分明るい） | `current`（補正なし） | 110 → 110 |

**Step C: パラメータの逆算** (`auto_grade_params()`)

```python
intensity = min(gap / 30.0, 1.0)
shadow_ratio = max(0, min(1, (70 - mean) / 70))  # 暗いほど1に近い
mid_ratio = 1.0 - shadow_ratio

lift   = intensity * (0.055 + 0.095 * shadow_ratio)  # 暗い画像でLift重視
gamma  = intensity * (0.028 + 0.050 * mid_ratio)      # 明るい画像でGamma重視
offset = intensity * 0.020                              # 常に控えめ

# 彩度: 標準偏差が低い（色が出ていない）ほど強く補正
sat_factor = 1.0 - min(std / 30, 0.5)
sat_boost = intensity * 0.10 * sat_factor
```

### 10.3 彩度の自動適応（実装済み）

彩度補正は `auto_grade_params()` 内で統合的に算出されます：

```python
# 元の彩度（標準偏差）が低い = 暗くて色が出ていない → 強めに補正
# 元の彩度が高い = 色が十分出ている → 控えめに
sat_factor = 1.0 - min(stats['std'] / 30, 0.5)
sat_boost = intensity * 0.10 * sat_factor
```

これにより：
- **暗くて彩度が低い写真**: 最大10%まで彩度を補正
- **既に彩度が十分な写真**: 5%程度に抑制
- **十分明るい写真**: 補正なし

### 10.4 肌スムージング（実装済み）

SegFormerの「肌」クラス（クラス1）のみを使い、**目・眉・唇を除外**した精密なマスクに対してバイラテラルフィルタを適用：

- **フィルタ径**: 画像サイズに応じて自動調整（`max(h,w) // 500`）
- **sigma_color/space**: strength=0.4で52（自然な滑らかさ）
- **適用範囲**: 肌領域のみ（マスク値 < 0.05 は完全除外）

### 10.5 スキントーンベクトルの適応性（実装済み）

`preserve_skin_tone_vector()` は写真ごとに肌色の方向を自動検出し、色補正がその方向から逸脱しないよう制約します：

| 被写体 | a,b平均 | 算出角度 | 機能 |
|--------|--------|---------|------|
| 今回の写真（日本人男性） | (3.8, 9.4) | 68.3° | 黄色寄りの暖色方向を維持 |
| 白人女性（仮定） | (5.0, 6.0) | 50.2° | ピンク寄りの方向を維持 |
| 褐色の肌（仮定） | (8.0, 15.0) | 61.9° | 深い暖色方向を維持 |
| 東南アジア人（仮定） | (4.0, 12.0) | 71.6° | 黄色味の強い方向を維持 |

**今後の改良ポイント**: 1枚の写真に複数人が写っていて肌色が異なる場合、人物ごとにスキントーン角度を計算する仕組みが必要です。これはSegFormerのセグメンテーションマップで個別の顔領域を分離することで実現可能です。

### 10.6 SaaS化の実現可能性まとめ

| 要素 | 現状 | SaaS化に必要な追加変更 | 難易度 |
